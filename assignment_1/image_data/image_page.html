3. Key Insights

  • ✓

    Inconsistent resolution: Although all images have the same Width, which is 150 pixels, and their height is mostly also 150 pixels but there are some a some number of images with smaller height.

  • ✓

    Class mostly balance: Even though the classes in this dataset are not perfectly balanced across all train, eval and testing dataset, with the largest class is mountain (or glacier) and smallest being building. But the differences are small and can be ignored. So the classes in this dataset are mostly ballance

  • ✓

    Sharpness and Contrast relationship: Most of the image have near zero or low sharpness, even with high contrast, indicate that most image going to have blurry, noisy background, which is most likely the blurry background when the image focus solely on the image

  • ✓

    Class correlation: Some classes are highly correlated to each other (for example Glacier and mountain and building and street), which is evident by the fact that their clustering is heavily overlapped with each other. This can be explain be the fact that the main object in this class can be present in other classes (like building can appear in street image, glacier can have mountain its image and vice versa)

  • ✓

    Task: Intel image classification is tailored to classification task, so there are no bounding box, segmentation mask or any annotation provided for any other specific task like segmentation and detection (unless manually added or automatically using pretrained model)
